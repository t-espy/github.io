RAG Document QA – Full Stack Bedrock Integration

This project is a full-stack demo of Retrieval-Augmented Generation (RAG) powered by a modern development stack: Python + FastAPI for the API layer, React + Vite for the frontend, and Amazon Bedrock (Titan models) for both embeddings and text generation.

Document ingestion, chunking, and indexing are performed locally via scripts that process files placed in the datastore/ directory and build a GPU-accelerated FAISS index using Bedrock embeddings. The resulting index is used to serve real-time document QA via REST APIs, with the entire stack deployable either locally or in the cloud.

Key Features

  • End-to-End Stack: Full RAG pipeline combining frontend, backend, embedding, retrieval, and generation layers
  • Modern UI: Built with React + Vite for fast dev cycles and responsive interaction
  • API Layer: FastAPI handles query routing, vector retrieval, and prompt orchestration with Bedrock
  • Document Processing: Scripts extract, chunk, and embed documents from datastore/
  • FAISS Index: Local GPU-accelerated semantic search using FAISS (flat or HNSW)
  • Amazon Bedrock: Titan Embeddings and Titan Text models used for inference (can be swapped)

Deployment Options

  • Docker Local Dev: Run the full stack locally with Docker Compose
  • AWS Deployment: Deploy via manual scripts or CloudFormation template
  • API-UI Integration: Deployed frontend calls live Bedrock-powered API to serve answers from uploaded documents

Technologies & Skills Demonstrated

  • Retrieval-Augmented Generation with FAISS and Bedrock Titan models
  • API development with FastAPI, LangChain, and Pydantic
  • Frontend engineering with React, Vite, and Tailwind CSS (optional)
  • Docker-based dev and deployment workflows
  • AWS cloud infrastructure and serverless orchestration
  • Prompt engineering and semantic chunk retrieval
